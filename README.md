# Wedding Invites — платформа свадебных приглашений

Telegram-бот + Web App: пара заполняет данные, админ подтверждает оплату, платформа создаёт страницу-приглашение вида `project.uz/ali-and-zebo` (+ именные ссылки `project.uz/ali-and-zebo/aziz` для премиум).

## Как это работает

1. Пара пишет боту `/start` → кнопка открывает Web App.
2. В форме: имена, дата (клик по календарю), время, локация на карте (ссылки на Google/Yandex Maps генерируются автоматически), минимум 2 фотографии пары, музыка, шаблон с ценой и живым примером (4 дизайна: Klassik, Bahor, Zumrad, Oltin Lux), опция «именные приглашения» (до 20 гостей).
3. Заявка сохраняется в базу, админу приходит сообщение с кнопками **✅ Оплачено** / **❌ Отклонить**.
4. Админ связывается с парой; после оплаты жмёт «Оплачено» → создаётся страница, пара получает ссылку в Telegram.

## Запуск

Нужен Node.js ≥ 22.5 (используется встроенный `node:sqlite`).

```bash
npm install
copy .env.example .env   # и заполнить BOT_TOKEN
npm start
```

### Настройка

1. **Токен бота** — создайте бота у [@BotFather](https://t.me/BotFather), токен в `.env` → `BOT_TOKEN`.
2. **ADMIN_CHAT_ID** — запустите сервер и напишите боту любое сообщение: пока id не настроен, бот ответит вашим chat id. Впишите его в `.env` и перезапустите.
3. **HTTPS** — Telegram Web App работает только по HTTPS. Для разработки:
   ```bash
   ngrok http 3000
   ```
   Полученный адрес (`https://xxxx.ngrok-free.app`) впишите в `.env` → `BASE_URL` и перезапустите. Для продакшена — свой домен (например `https://project.uz`) через reverse proxy (Caddy/Nginx).

### Разработка без Telegram

`DEV_NO_AUTH=1` в `.env` — форма открывается в обычном браузере: http://localhost:3000/app/ (заявка создастся от имени пользователя `dev`, id 0). **Не включать в продакшене** — это отключает проверку подписи Telegram.

## Структура

```
src/
  index.js     точка входа: запускает бота (long polling) и сервер
  config.js    .env + шаблоны/цены/треки (менять цены здесь)
  db.js        SQLite (data/wedding.db), таблицы applications и guests
  service.js   бизнес-логика: валидация заявки, подтверждение оплаты, slug-и
  slug.js      транслитерация (ru/uz кириллица) и генерация slug
  initData.js  проверка подписи Telegram Web App
  bot.js       /start, кнопки админа, уведомления
  server.js    Express: форма, API (upload, applications), страницы приглашений
  render.js    диспетчер шаблонов + подготовка данных (buildData, renderDemo)
  upload.js    сохранение фото (data/uploads), тип по сигнатуре файла
  templates/   4 шаблона открыток: klassik, bahor, zumrad, oltin
public/
  app/         Web App форма (без сборки: HTML+CSS+JS, свой календарь)
  demo/        sample-фото для демо; /demo/<id> рендерится из шаблонов
  music/       mp3 для готовых треков (см. MUSIC_PRESETS в config.js)
data/          база SQLite и загруженные фото (в git не попадает)
```

Новый шаблон: файл в `src/templates/`, импорт в `src/render.js` (RENDERERS) и запись в `TEMPLATES` в `src/config.js` — демо и страницы подключатся автоматически.
